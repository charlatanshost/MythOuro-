# Dataset Selection — vetting rubric & decision log

**Purpose.** The *rules and rationale* for choosing training data, plus a running
**accept / reject / pending** log so decisions stay consistent and rejected datasets
aren't re-evaluated. The specific-dataset registry lives in
[`clean_sft_datasets.md`](clean_sft_datasets.md); what's *currently used* is in
[`datasets.md`](datasets.md); license consequences are in the README "Licensing &
data provenance" section. **This file is the "how we decide."**

> **The one lesson behind all of it (2026-06-20):** *verify the generating model,
> not the source.* MIRIAD looked clean ("grounded in S2ORC") but was **generated by
> OpenAI's models** — disqualifying. Removing OpenHermes was necessary but not
> sufficient. **Read the card + paper; assume nothing.**

---

## Hard constraints (gates — a dataset must pass ALL of these to be eligible)

1. **No OpenAI-output provenance.** No data generated by GPT-3.5/4/4o or sourced from
   ChatGPT/GPT outputs (incl. user↔GPT logs like WildChat/ShareGPT). OpenAI's terms
   restrict using outputs to train competing models → gates distribution. **Check the
   *generator*, not just the input/grounding.**
2. **No Claude/Anthropic-output provenance.** Same logic, for consistency (we also do
   not generate training data with Claude — see README dev-tooling note; Anthropic
   Consumer-Terms clarification pending).
3. **Redistributable license** (for an open/distributable dataset): CC-BY, CC0,
   Apache-2.0, MIT, ODC-By. **Avoid** CC-BY-**NC** (non-commercial) if commercial-clean
   is wanted, **ND** (no-derivatives — can't even make a derived dataset), and
   **credential-gated** clinical data (MIMIC-III/IV, i2b2, eICU — DUA-bound, **cannot
   be redistributed**).
4. **Generator-license clean** (if model-generated): the *generating* model's license
   must permit using its outputs to train other models — **Apache models (Qwen,
   Mistral) and Llama 3.1+ are OK**; avoid generators that restrict output reuse.
5. **No benchmark contamination** with our eval sets (GSM8K, ARC, etc.) — run
   `data/contamination.py`; drop overlapping samples.

---

## ✅ Green flags (what to look for)

- **Human-authored** (no model generation) — cleanest possible provenance.
- **Generated by a *disclosed open* model** (Llama 3.1+/Qwen/Mistral), generator named.
- **Verification loops** — code with unit-test execution status; math with checked answers.
- **Grounded generation** (RAG-style: generated *from* a cited source passage) — factual,
  citable, low-hallucination. Keep the citation with each item.
- **Permissive, clearly-stated license** (CC-BY/CC0/Apache/MIT/ODC-By), ideally per-item.
- **Documented provenance** — the card/paper names the generator and the source.

## ❌ Red flags (what to avoid / treat as suspect)

- **"Generated with GPT-4 / GPT-4o / ChatGPT / OpenAI"** anywhere in the pipeline → reject.
- **Undisclosed generator** — "synthetic"/"hybrid" with no model named → **suspect until
  verified** (don't assume open). Read the linked paper.
- **GPT-distilled instruction sets** — Alpaca, many "*-Evol-Instruct"/"*-Instruct" sets,
  OpenHermes, ShareGPT, WildChat-GPT — all OpenAI-output.
- **Misleading wording:** "no proprietary model *knowledge*" (describes grounding, not the
  generator — MIRIAD's exact dodge); "open dataset" (license ≠ provenance);
  "research-friendly" (often = NC or murky).
- **NC / ND licenses** if distributing; **credential-gated** clinical data (can't redistribute).
- **Benchmark overlap** with our evals (contamination).

---

## Vetting procedure (run for every candidate)

1. **Find the generating model** — read the HF card *and* the linked paper. If it's not
   stated, it's **pending**, not clean.
2. **Check the license** — per-item if the set is mixed; confirm it's in the
   redistributable set above.
3. **Check generator license** (if model-made) — outputs reusable for training?
4. **Contamination scan** vs. our eval benchmarks.
5. **Record the decision below** (accept / reject / pending + the reason).

## Building our own (the clean medical path)

Since MIRIAD is out and medical is the core domain, the most defensible source is
**self-built from the PMC Open Access Subset:**
- Pull the **`oa_comm`** bucket (commercial-use OK: CC-BY/CC0) via NCBI FTP
  (`ftp.ncbi.nlm.nih.gov/pub/pmc/`) or the OA Web Service — **not** the web reader
  (free-to-read ≠ redistributable). NCBI pre-sorts by license, so `oa_comm` is the
  clean bucket.
- Parse the **JATS XML** (`pubmed_parser` / lxml) into passages; keep **license +
  citation** per article.
- **Grounded generation** with an **open model** (Qwen/Mistral/Llama-3.1+) — generate QA
  *from the passage*, not from the model's memory. Verify + dedup + contamination-scan.
- **Dual-use:** the same `oa_comm` corpus is also the future MemPalace/RAG retrieval store.
