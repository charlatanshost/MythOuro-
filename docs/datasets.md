These are the corpora the distillation + SFT pipeline currently consumes
(see [`mythouro/training_utils.py`](../mythouro/training_utils.py) for the
distillation mix and [`mythouro/sft_data.py`](../mythouro/sft_data.py) for SFT).

| Dataset | HuggingFace | Stage | License (nominal) | Provenance caveat |
|---|---|---|---|---|
| FineWeb-Edu | [HuggingFaceFW/fineweb-edu](https://huggingface.co/datasets/HuggingFaceFW/fineweb-edu) | distill (general) | ODC-By 1.0 | Web text; standard CommonCrawl provenance |

OpenAI ToS restricts using outputs to train competing models. The strongest provenance flag in the mix. |

> **Provenance & licensing note (important if distributing checkpoints).**
> The three SFT corpora (OpenHermes 2.5, MetaMathQA, Magicoder-Evol-Instruct)
> all contain data **generated by OpenAI models**. For a personal research
> project this is low practical risk, but if you ever **distribute trained
> weights or use them commercially**, the OpenAI-output provenance is a real
> gating consideration — OpenAI's terms restrict using their outputs to train
> competing models, and the "license" tags on these HF datasets do not override
> that upstream constraint. The distillation teacher (`ByteDance/Ouro-2.6B-Thinking`)
> is cleanly **Apache 2.0** and permits distillation/redistribution with
> attribution. See the "Licensing & data provenance" section of the README.
>
> To produce a checkpoint with clean provenance, swap the SFT mix to corpora
> without OpenAI-generated content (e.g. Tulu-3 SFT subsets that document their
> provenance, Dolly-15k (human-written), or self-generated data from an
> Apache/MIT-licensed model).

## Primary Pretraining

### FineWeb-Edu
- **HuggingFace:** `HuggingFaceFW/fineweb-edu`
- **Size:** 1.3T tokens
- **License:** ODC-By 1.0 (CommonCrawl-derived)
- **Why:** Web text filtered for educational quality. Outperforms The Pile, C4, and RefinedWeb on downstream benchmarks. Already deduplicated and cleaned.
- **Start with:** `sample-10BT` to validate your pipeline, then `sample-100BT` or the full corpus for a serious run.

## Supplementary

### OpenHermes 2.5
- **HuggingFace:** `teknium/OpenHermes-2.5`
- **Size:** ~1M instruction samples
- **License:** Mixed — a *compilation* of many sub-datasets, some containing
  GPT-4-generated data. The dataset card does not assert a single clean license;
  treat as **OpenAI-output provenance** (see the provenance note above), not
  "Apache 2.0".
- **Why:** High-quality instruction-following data. Used in the SFT mix
  (~30%) at `seq_len≥1024` — at 512 its multi-turn conversations are ~95%
  rejected by the length filter.

### OpenWebMath
- **HuggingFace:** `open-web-math/open-web-math`
- **Size:** ~14.7B tokens
- **License:** ODC-By
- **Why:** Math-focused web text. Add if you want stronger quantitative and symbolic reasoning. Particularly useful for the 10B+ variants where reasoning depth matters.

## Token Budget Recommendations

| Variant | Chinchilla-optimal | Recommended (looped) |
|---|---|---|
| 1B | ~20B tokens | ~10–15B tokens |
| 3B | ~60B tokens | ~30–40B tokens |
| 10B | ~200B tokens | ~100–150B tokens |
| 50B+ | ~1T+ tokens | ~500B+ tokens |

The looped architecture is more sample-efficient than a standard transformer — same validation loss is reachable with fewer tokens due to faster convergence. The "recommended (looped)" column reflects this and is based on the Tiny Shakespeare result where MythOuro reached equivalent loss ~2.5× faster than nanoGPT.
